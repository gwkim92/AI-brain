# JARVIS 세계모델 코어 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 기존 JARVIS의 `session -> dossier -> watcher -> action proposal` 흐름 안에 세계모델 코어를 삽입해, 리서치 결과를 `가설/반대가설/병목/무효화 조건`까지 추적 가능한 구조로 확장한다.

**Architecture:** 별도 신규 제품을 만들지 않는다. Postgres 중심 저장 구조 위에 `candidate fact -> semantic normalization -> state update -> hypothesis ledger -> dossier/monitor/action enrichment` 파이프라인을 얹고, 기존 Assistant/Dossier/Watchers/Action Center를 그대로 강화한다. V1에서는 그래프 DB를 도입하지 않고 `Postgres + JSONB + typed tables`로 시작한다.

**Tech Stack:** TypeScript, Fastify, Next.js 16, Postgres, JSONB, pgvector, Vitest, Playwright, Zod

## 권장 실행 순서

반드시 아래 순서로 진행한다.

1. 저장소 계약과 스키마
2. 추출과 정규화
3. 상태 모델과 가설 원장
4. dossier/briefing enrichment
5. watcher/action proposal 연결
6. eval/outcome journal
7. feature flag와 회귀 고정

이 순서를 바꾸면 안 되는 이유:

- 저장 계약 없이 추론 로직부터 만들면 이후 마이그레이션 비용이 급증한다
- dossier UI부터 손대면 근거 없는 서사 UI만 먼저 생긴다
- watcher 후속판정을 먼저 바꾸면 현재 제품의 proactive precision을 해칠 위험이 크다
